# Contributing to MESSAGE-IX Two-Region Energy System Model

Thank you for your interest in contributing to this energy system modeling project! This guide outlines how to contribute effectively.

## Code of Conduct

This project adheres to academic and professional standards. Contributors are expected to:
- Maintain respectful and constructive communication
- Follow scientific rigor in methodology and documentation
- Acknowledge sources and collaborations appropriately
- Respect intellectual property and licensing terms

## How to Contribute

### Reporting Issues
- Use the issue templates provided
- Include detailed information about your environment
- Provide reproducible examples when possible
- Search existing issues before creating new ones

### Suggesting Enhancements
- Use the feature request template
- Describe the use case and motivation
- Consider implementation feasibility
- Discuss potential impacts on model accuracy

### Code Contributions

#### Development Setup
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Install development dependencies: `pip install -r requirements.txt`
4. Make your changes
5. Test your changes thoroughly
6. Submit a pull request

#### Coding Standards
- Follow PEP 8 style guidelines for Python code
- Include docstrings for all functions and classes
- Add unit tests for new functionality
- Update documentation as needed
- Ensure compatibility with MESSAGE-IX framework standards

#### Documentation Standards
- Use clear, professional language
- Include mathematical formulations where appropriate
- Cite relevant sources and methodologies
- Update README.md for significant changes
- Maintain technical documentation in `docs/` directory

### Types of Contributions Welcomed

#### Model Extensions
- Additional energy technologies (hydrogen, nuclear, biomass, etc.)
- Enhanced temporal resolution (sub-annual, hourly optimization)
- Spatial extensions (multi-regional transmission modeling)
- Coupling with other sectors (transport, heating, industry)

#### Data Improvements
- Updated technology cost databases
- Regional resource assessments
- Policy scenario development
- Validation against real-world systems

#### Methodological Enhancements
- Advanced optimization algorithms
- Uncertainty quantification methods
- Sensitivity analysis frameworks
- Model validation techniques

#### Educational Resources
- Tutorial notebooks
- Classroom exercises
- Case study development
- Video documentation

## Academic Use Guidelines

### Citation Requirements
If you use this model in academic work, please cite the repository and acknowledge key data sources:
- NREL Annual Technology Baseline
- IEA World Energy Outlook
- IPCC Assessment Reports
- MESSAGE-IX framework (IIASA)

### Collaboration Opportunities
We welcome academic collaborations including:
- Joint research publications
- Student thesis projects
- Course integration
- Conference presentations

### Data Sharing
Contributors are encouraged to share:
- Regional adaptation datasets
- Validation case studies
- Sensitivity analysis results
- Policy scenario specifications

## Review Process

### Pull Request Guidelines
1. **Description**: Provide clear description of changes and motivation
2. **Testing**: Include test results and validation checks
3. **Documentation**: Update relevant documentation
4. **Breaking Changes**: Clearly identify any breaking changes
5. **Performance**: Consider computational performance impacts

### Review Criteria
- Scientific accuracy and methodology soundness
- Code quality and maintainability
- Documentation completeness
- Compatibility with existing functionality
- Test coverage and validation

### Feedback Process
- Reviewers will provide constructive feedback
- Authors should address comments thoroughly
- Multiple review rounds may be necessary
- Final approval required from repository maintainers
